# Perfetto SQL 可视化分析工具

## 你的角色
你是一名资深前端架构师 + 数据分析工具设计师，正在帮助我设计并实现一个面向 Perfetto trace 分析的可视化 SQL 工具。

你需要在理解 Perfetto trace 数据结构的基础上，把常用 SQL 查询抽象成可视化插件，让用户不用手写复杂 SQL 也能完成常见分析任务。

---

## 需求背景
Perfetto 官网 ui.perfetto.dev 的 SQL 模块虽然功能强大，但对普通用户不够友好，主要存在以下问题：

1. 用户不知道 trace 中有哪些表、每个表有哪些字段
2. SQL 编写门槛高，学习成本大，效率低
3. 常见分析场景每次都要手写重复 SQL，非常浪费时间
4. 用户很难通过 SQL 直接得到图表化结果

因此，我希望基于 Perfetto SQL 做二次开发，构建一个 **“常用分析能力可视化 + SQL 插件化 + 可扩展”** 的分析工具。

---

## 项目目标
请围绕以下目标设计和实现产品：

1. 将 Trace 数据转换或映射成可查询的数据库结构
2. 内置一批常用的分析插件，用户可以一键查询
3. 支持用户动态新增 SQL 插件
4. 提供可视化结果展示能力
5. 降低用户对 SQL 和 Perfetto 表结构的理解成本
6. 让用户可以通过表单式配置完成分析，而不是直接写 SQL

---

## 核心思路
请优先采用以下产品思路：

### 1. 数据层
- 将 trace 数据解析后，映射为数据库可查询结构
- 自动识别常用表、字段、索引、时间范围等信息
- 支持查看表结构、字段说明、示例数据
- 支持按 schema 生成字段元信息，方便 UI 展示和智能补全

### 2. 插件层
- 将常用分析能力封装为“SQL 插件”
- 每个插件本质上对应一个可配置的查询模板
- 插件可通过表单参数动态生成 SQL
- 插件支持扩展、编辑、保存、复用

### 3. 可视化层
- 将查询结果自动渲染成合适的图表或表格
- 支持折线图、柱状图、饼图、明细表、统计卡片等
- 对时间序列类结果优先支持折线图
- 对列表类结果优先支持表格
- 对聚合结果优先支持统计卡片

## 功能需求

### 1. Trace 导入
- 支持导入 trace 文件

  - 记录条数

### 2. Trace 分析插件
请预置以下高频分析插件，并为每个插件设计可复用的参数配置方式：

#### 插件 A：某段时间内，某个进程、某个线程下，满足模糊匹配的 slice 列表
示例：
- 0~10 秒内
- target 进程
- 主线程
- 名称包含 `Activity`
- 输出该时间段内匹配到的所有 slice 列表

输出要求：
- 支持按时间排序
- 显示 ts / dur / name / thread / process 等字段
- 支持点击查看详情

---

#### 插件 B：某段时间内，某个进程的线程数量变化趋势
示例：
- 0~10 秒内
- target 进程
- 统计线程数量随时间的变化
- 输出折线图

输出要求：
- 支持时间分桶
- 支持按秒/毫秒粒度切换
- 支持展示峰值、均值、最大值、最小值

---

#### 插件 C：某段时间内，某个进程、某个线程、某类事件的总耗时
示例：
- 0~10 秒内
- target 进程
- 主线程
- `*doFrame*` 函数总耗时

输出要求：
- 支持模糊匹配
- 支持按事件名分组统计
- 支持总耗时、平均耗时、次数统计

---

#### 插件 D：某段时间内，某个进程、某个线程下，指定事件的执行时间戳
示例：
- 0~10 秒内
- target 进程
- 主线程
- `onCreate / onStart / onResume` 三个事件的执行时间戳

输出要求：
- 支持多个事件名同时查询
- 支持按照执行顺序展示
- 支持导出结果
- 支持在时间轴上高亮标记

---

#### 插件 E：后续可扩展的通用分析插件
请预留扩展能力，支持后续增加：
- CPU 分析
- 调度分析
- Binder 分析
- frame/jank 分析
- 内存分析
- IO 分析
- counter 曲线分析
- 自定义聚合分析

## 可视化能力
请实现一个统一的结果展示层，至少支持以下类型：

1. **表格**
