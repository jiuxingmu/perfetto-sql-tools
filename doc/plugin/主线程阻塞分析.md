# 主线程阻塞分析

## 1. 插件名称
主线程阻塞分析

## 2. 插件目标
识别主线程阻塞事件、唤醒上下文和可疑阻塞片段，辅助卡顿根因定位。

## 3. 适用场景
- 卡顿与掉帧排查
- 主线程阻塞来源分析
- IO wait 与不可中断睡眠分析

## 4. 输入参数
### 必填
- `start_ts`
- `end_ts`
- `process_name`

### 可选
- `suspicious_only`（默认 `true`）

## 5. 核心功能
### 5.1 阻塞片段提取
提取窗口内主线程 blocked/sleeping 相关状态片段。

### 5.2 可疑阻塞识别
按规则识别疑似阻塞：
- `state = D`
- `io_wait = 1`
- `dur >= 7ms`

### 5.3 唤醒链路展示
输出唤醒线程和唤醒进程信息，辅助上下文分析。

## 6. 页面展示要求
### 6.1 筛选区
- 时间范围
- 进程
- 疑似阻塞过滤开关

### 6.2 结果摘要
- 当前结果总数
- 疑似阻塞口径说明

### 6.3 图表区域
- 可选：阻塞时长 Top N 柱状图

### 6.4 明细表格
建议字段：
- `blocked_start_ts_sec`
- `blocked_end_ts_sec`
- `blocked_dur_ms`
- `blocked_state`
- `blocked_reason`
- `waker_tid`
- `waker_thread`
- `waker_process`

## 7. 交互要求
- 支持重新查询
- 支持疑似/全量口径切换
- 支持导出结果

## 8. 输出数据要求
- 汇总数据：总数与口径说明
- 明细数据：阻塞事件列表
- 原始结果：用于深入复核

## 9. 核心逻辑说明
### 9.1 统计方式
先确定目标主线程，再在窗口内筛选 thread_state 片段。

### 9.2 聚合规则
事件粒度逐条输出，不做二次聚合。

### 9.3 排序规则
按阻塞时长降序，再按开始时间升序。

## 10. 验收标准
- 主线程选择逻辑稳定
- 疑似过滤口径正确
- 阻塞与唤醒信息可用于定位问题
