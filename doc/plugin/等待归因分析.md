# 等待归因分析

## 1. 插件名称
等待归因分析

## 2. 插件目标
帮助用户分析线程阻塞或等待时“到底在等什么”，并尽可能归因到可执行的问题类型。

## 3. 适用场景
- 主线程假死分析
- D 状态分析
- 锁竞争排查
- IO 卡顿排查
- 系统调用阻塞分析
- trace 中等待片段归因

## 4. 输入参数
### 必填
- `start_ts`
- `end_ts`

### 可选
- `process_name`
- `pid`
- `thread_name`
- `tid`
- `blocked_threshold_ms`（默认 5）
- `wait_type_filter`（`io/lock/binder/futex/workqueue/schedule`）
- `only_main_thread`（默认 true）

## 5. 核心功能
### 5.1 等待片段识别
识别窗口内线程阻塞/等待片段。

### 5.2 等待原因归因
将等待片段归类到 IO、锁、Binder、futex、workqueue、schedule 或 unknown。

### 5.3 等待统计
统计累计时长、次数、平均时长、最长时长、类型占比。

### 5.4 关联上下文
尽量关联 blocked function、waker 线程/进程、时间上下文。

## 6. 页面展示要求
### 6.1 筛选区
- 时间范围
- 进程 / 线程
- 阈值设置
- 等待类型筛选
- 仅主线程

### 6.2 结果摘要
- 总等待时长
- 等待次数
- 最长等待时长
- 最主要等待类型
- IO/锁/Binder 占比

### 6.3 图表区域
- 饼图：等待类型占比
- 柱状图：类型累计时长
- 时间线图：等待区间分布

### 6.4 明细表格
- 等待开始时间 / 结束时间 / 等待时长
- 等待类型 / 阻塞状态 / 阻塞函数
- 唤醒线程 / 唤醒进程
- 备注（blocked_reason）

## 7. 交互要求
- 点击记录可展开详情
- 支持查看 waker 信息
- 支持图表与表格联动高亮
- 支持重新查询
- 支持导出结果

## 8. 输出数据要求
### 8.1 汇总数据
用于摘要卡片与图表展示。

### 8.2 明细数据
用于等待记录表格展示。

### 8.3 原始结果
用于定位与导出。

建议字段：
- `blocked_reason`
- `blocked_state`
- `blocked_start_ts`
- `blocked_end_ts`
- `blocked_dur_ms`
- `wait_type`
- `blocked_function`
- `waker_thread`
- `waker_process`
- `process_name`
- `pid`
- `tid`

## 9. 核心逻辑说明
### 9.1 阻塞识别
基于线程状态片段识别阻塞区间并按阈值过滤。

### 9.2 等待归类
按函数特征、状态特征、waker 上下文进行规则归类。

### 9.3 阈值过滤
仅保留持续时长超过阈值的片段。

### 9.4 排序规则
按等待时长降序。

## 10. 验收标准
- 能识别等待片段
- 能进行等待类型归因
- 能统计占比与累计时长
- 能展示阻塞与唤醒信息
- 查询结果与 SQL 口径一致

## 11. 与其他插件边界
- 本插件回答“为什么在等”（原因层）：IO/锁/Binder/futex/workqueue/schedule 分类。
- 不负责慢帧判定和卡顿区间定位；这部分由`慢帧分析`负责。
- 主线程阻塞证据（blocked 规则命中、可疑阻塞口径）由`主线程阻塞总览`补充验证。
