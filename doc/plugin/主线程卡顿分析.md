# 慢帧分析

## 1. 插件名称
慢帧分析

## 2. 插件目标
帮助用户快速识别主线程在指定时间范围内是否发生卡顿，并明确卡顿区间、慢帧情况和主要执行上下文。

## 3. 适用场景
- 页面掉帧分析
- UI 卡顿排查
- 主线程阻塞排查
- 用户交互延迟分析
- trace 中慢帧定位

## 4. 输入参数
### 必填
- `start_ts`
- `end_ts`

### 可选
- `process_name`
- `pid`
- `thread_name`
- `tid`
- `frame_threshold_ms`（默认 16.6）
- `slow_frame_threshold_ms`（默认 33）
- `only_main_thread`（默认 true）

## 5. 核心功能
### 5.1 慢帧识别
识别主线程中耗时超过阈值的执行片段。

### 5.2 卡顿区间定位
输出主线程明显超时的时间段。

### 5.3 卡顿原因关联
关联卡顿期间的 `slice` 名称、线程状态与阻塞原因提示。

### 5.4 指标统计
- 帧总数
- 慢帧数量
- 慢帧占比
- 平均帧耗时
- 最大帧耗时
- 最严重卡顿时间段

## 6. 页面展示要求
### 6.1 筛选区
- 时间范围
- 进程
- 线程
- PID / TID
- 帧阈值
- 慢帧阈值
- 是否仅主线程

### 6.2 结果摘要
- 总帧数
- 慢帧数
- 慢帧占比
- 最大卡顿时长
- 最严重时间段

### 6.3 图表区域
- 卡顿时间线图
- 帧耗时分布图
- 慢帧占比图

### 6.4 明细表格
- 开始时间 / 结束时间 / 持续时间
- 是否慢帧 / 卡顿类型
- 主线程状态 / 阻塞原因
- 关联 slice/函数名

## 7. 交互要求
- 点击记录可展开详情
- 图表与表格联动高亮
- 支持重新查询
- 支持导出结果

## 8. 输出数据要求
### 8.1 汇总数据
用于摘要卡片与图表。

### 8.2 明细数据
用于慢帧列表展示。

### 8.3 原始结果
用于排查与导出。

建议字段：
- `frame_start_ts`
- `frame_end_ts`
- `frame_dur_ms`
- `slow_flag`
- `main_thread_state`
- `blocking_reason`
- `top_slice_name`
- `process_name`
- `pid`
- `tid`

## 9. 核心逻辑说明
### 9.1 慢帧判断
按阈值判断 `frame_dur_ms` 是否为慢帧。

### 9.2 卡顿识别
对主线程执行片段按耗时排序并标注卡顿类型。

### 9.3 归因方式
优先关联：
- `slice` 名称
- 线程状态
- 阻塞原因
- 前后上下文

### 9.4 排序规则
按持续时长（帧耗时）从高到低。

## 10. 验收标准
- 能识别主线程慢帧
- 能展示卡顿时间段和持续时间
- 能展示摘要、图表、表格
- 能关联卡顿对应执行上下文
- 查询结果与 SQL 统计逻辑一致

## 11. 与其他插件边界
- 本插件回答“哪里卡了”（现象层）：慢帧、卡顿区间、严重程度。
- 不负责完整等待归因；若要回答“为什么在等”，使用`等待归因分析`。
- 若要验证主线程是否存在明确阻塞证据与唤醒链路，使用`主线程阻塞总览`。
