# CPU 占用分析

## 1. 插件名称
CPU 占用分析

## 2. 插件目标
帮助用户在指定时间范围内快速识别 CPU 热点对象，回答“谁最忙、忙了多久、占比多少”。

## 3. 适用场景
- 页面卡顿排查
- ANR 排查
- 线程抢占分析
- CPU 异常升高分析

## 4. 输入参数
### 必填
- `start_ts`：开始时间
- `end_ts`：结束时间

### 可选
- `process_name`：进程名
- `pid`：进程 ID
- `stat_level`：统计粒度（`process` / `thread`）
- `top_n`：默认 10
- `only_main_thread`：是否仅主线程，默认 `false`

## 5. 核心功能
### 5.1 CPU 占用统计
统计窗口内的 CPU 累计时长、占比、执行片段数、平均单段时长。

### 5.2 统计维度切换
- 进程维度：按 `pid` 聚合
- 线程维度：按 `tid` 聚合

### 5.3 Top N 排序
按 `cpu_dur_ms` 降序输出前 N 条热点。

## 6. 页面展示要求
### 6.1 筛选区
- 时间范围
- 进程
- PID
- 统计粒度
- Top N
- 仅主线程

### 6.2 结果摘要
- 总 CPU 时长
- Top 1 对象
- 热点线程数
- Top10 占比

### 6.3 图表区域
- 柱状图：Top N CPU 占用对象

### 6.4 明细表格
字段建议：
- `name`
- `pid` / `tid`
- `cpu_dur_ms`
- `cpu_ratio`
- `slice_count`
- `avg_slice_dur_ms`

## 7. 交互要求
- 支持重新查询
- 支持导出结果
- 支持图表与表格联动高亮

## 8. 输出数据要求
### 8.1 汇总数据
用于摘要卡片与图表展示。

### 8.2 明细数据
用于表格展示。

### 8.3 原始结果
用于 SQL 预览与问题排查。

## 9. 核心逻辑说明
### 9.1 统计方式
基于调度片段与时间窗口裁剪后计算 CPU 时长。

### 9.2 聚合规则
- `process`：按进程聚合
- `thread`：按线程聚合

### 9.3 排序规则
按 `cpu_dur_ms` 从高到低排序。

## 10. 验收标准
- 能输出窗口内 CPU 热点
- 能切换进程/线程维度
- 能正确输出 Top N
- 能展示摘要、图表、表格
