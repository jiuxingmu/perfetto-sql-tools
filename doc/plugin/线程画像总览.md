# 线程画像总览

## 1. 插件名称
线程画像总览

## 2. 插件目标
在线程维度提供“先结论后明细”的分析入口，帮助用户快速识别值得优先排查的线程。

## 3. 适用场景
- 主线程异常排查
- 线程热点筛选
- 调度与唤醒频繁问题定位

## 4. 输入参数
### 必填
- `start_ts`
- `end_ts`

### 可选
- `process_name`
- `thread_name`
- `pid`
- `tid`
- `only_main_thread`
- `only_active`
- `top_n`
- `sort_by`（`cpu_time` / `active_duration` / `switch_count` / `wakeup_count`）

## 5. 核心功能
### 5.1 线程画像指标
输出线程级指标：
- `active_duration_ms`
- `cpu_time_ms`
- `switch_count`
- `wakeup_count`
- `thread_type`

### 5.2 一句话结论
给出总线程、活跃线程、主线程数量及热点线程结论。

### 5.3 可视化 + 明细联动
提供 Top N 图和线程表格，支持快速筛查。

## 6. 页面展示要求
### 6.1 分析结论
- 线程总数
- 活跃线程数
- 主线程数
- 热点线程说明

### 6.2 线程画像图
- 柱状：CPU 时间
- 折线：切换次数、唤醒次数

### 6.3 线程明细表
- 线程名 / TID / PID / 进程
- 是否主线程 / 线程类型
- 活跃时长 / CPU 时间
- 切换次数 / 唤醒次数
- 建议下一步分析

## 7. 输出数据要求
- 汇总：总量与热点结论
- 明细：线程画像字段
- 原始：用于 SQL 复核与导出

## 8. 验收标准
- 能按筛选条件输出线程画像
- 排序规则正确且可切换
- 图表与表格数据一致
- 能快速识别主线程与热点线程
